# P2 JSON vs XML Testing

Repository ini berisi hasil **Praktikum 2 Web Service Engineering (WSE)** dengan topik
**perbandingan response data JSON dan XML** menggunakan Postman dan Browser DevTools.

## Tujuan Praktikum
1. Mengirim request HTTP (GET) dan membaca response API.
2. Mengidentifikasi perbedaan struktur data JSON dan XML.
3. Membandingkan ukuran (size) dan waktu respons (time) JSON dan XML.
4. Menganalisis implikasi penggunaan format data terhadap aplikasi, khususnya aplikasi mobile.

## Tools yang Digunakan
- Postman
- Browser (Chrome / Edge) dengan DevTools (Network Panel)
- API Publik

## Endpoint yang Diuji
### JSON

### XML

## Langkah Praktikum
1. Melakukan pengujian endpoint JSON menggunakan Postman.
2. Melakukan pengujian endpoint XML menggunakan Postman.
3. Mengamati status code, headers, response body, ukuran data, dan waktu respons.
4. Melakukan pengujian ulang melalui Browser DevTools (Network).
5. Mengambil screenshot sebagai bukti pengujian.
6. Membuat tabel perbandingan JSON dan XML.
7. Menyusun analisis singkat berdasarkan hasil pengujian.

## Hasil Singkat
- JSON memiliki struktur yang lebih sederhana dan mudah dibaca.
- Ukuran response JSON cenderung lebih kecil dibandingkan XML.
- Waktu respons JSON umumnya lebih cepat.
- XML memiliki struktur lebih kompleks dengan validasi yang lebih ketat.

## Kesimpulan
Berdasarkan hasil pengujian, format JSON lebih efisien dan cocok digunakan untuk aplikasi
modern seperti aplikasi web dan mobile karena lebih ringan dan cepat.  
Sementara itu, XML lebih sesuai digunakan pada sistem yang membutuhkan struktur data
yang ketat dan validasi formal.
